# SmartAgent / LearnAgent 项目功能模块分析文档

> 生成日期：2026-06-03  
> 基于 Spring Boot 3.3.5 + Spring AI 1.1.5 + Vue 3 + TypeScript

---

## 一、项目概览

**项目名称**：SmartAgent（智能体平台） / LearnAgent（个人学习助手）  
**技术架构**：前后端分离  
- **后端**：Java 17 + Spring Boot 3.3.5 + Spring AI 1.1.5 + MySQL + MongoDB + Redis + Qdrant  
- **前端**：Vue 3.5 + TypeScript + Element Plus + Pinia + Vite 6  
- **AI 模型**：DeepSeek（主模型） + Qwen（兜底模型）  
- **向量化**：阿里云 DashScope text-embedding-v4

**核心能力概括**：
1. 多种 AI Agent 模式（对话、工具调用、RAG 检索、任务规划、结构化输出）
2. 多轮记忆对话（基于 Redis/内存 + MessageChatMemoryAdvisor）
3. RAG 知识库（文档上传 → 切片 → 向量化 → 相似度检索 → 生成回答）
4. 工具调用（计算器、日期时间、翻译、文档解析、数据库查询）
5. 任务规划（AI 拆解复杂任务 → 逐步执行 → 汇总结果）
6. SSE 流式输出（所有 Agent 均支持流式）
7. LearnAgent 个人学习系统（ReAct 推理 + 笔记 + 测验 + 闪卡 + 知识图谱 + 学习统计）
8. 完整的管理后台（用户管理、审计日志、监控面板、权限控制）

## 二、后端功能模块详解

### 2.1 认证授权模块 (Auth)

**涉及文件**：
- `controller/AuthController.java`
- `auth/JwtTokenProvider.java`
- `auth/JwtAuthenticationFilter.java`
- `auth/JwtProperties.java`
- `auth/UserDetailsServiceImpl.java`
- `config/SecurityConfig.java`

**功能说明**：
- 用户注册（`POST /api/v1/auth/register`）：创建 USER 角色账号，自动返回 JWT Token
- 用户登录（`POST /api/v1/auth/login`）：验证密码（BCrypt），返回 JWT Token
- 管理员初始化（`POST /api/v1/auth/init-admin`）：仅 dev 环境可用，创建 ADMIN 角色账号
- JWT Token 生成：包含 username + role，默认过期时间可配置
- 鉴权流程：`JwtAuthenticationFilter` 从 `Authorization: Bearer <token>` 头提取 Token，验证后设置 `SecurityContext`

**执行逻辑**：
```
用户请求 → JwtAuthenticationFilter 提取 Token → 验证签名/过期 
→ 查询数据库确认用户存在且启用 → 设置 Spring Security Context 
→ SecurityConfig 根据角色匹配 URL 权限规则
```

**权限分级**：
| 角色 | 权限范围 |
|------|---------|
| 匿名 | 登录、注册、健康检查、Knife4j 文档 |
| USER | 对话、知识库问答、知识库检索、个人学习模块、首页、演示 |
| ADMIN | 用户管理、监控面板、审计日志、知识库上传/加载、运维面板、部署验收 |

### 2.2 对话 Agent 模块 (ChatAgent)

**涉及文件**：
- `agent/ChatAgent.java`
- `controller/ChatController.java`

**功能说明**：
- 多轮记忆对话（`POST /api/v1/chat/memory`）：带 conversationId 实现上下文记忆
- 流式记忆对话（`GET /api/v1/chat/memory/stream`）：SSE 流式推送
- 兜底模型切换：主模型（DeepSeek）失败时自动切换备用模型（Qwen）

**执行逻辑**：
```
用户消息 → resolveConversationId（空则生成 UUID）→ ChatClient.prompt()
  → MessageChatMemoryAdvisor 注入历史消息（最多50条）
  → 调用主模型 → 成功：返回回复
  → 失败：fallbackMemoryChat() 切换到备用模型
  → AuditService 保存聊天历史和调用记录
```

**关键设计**：
- `MessageChatMemoryAdvisor` 自动管理历史消息的加载与保存
- `conversationId` 实现会话隔离，不同会话的历史互不干扰
- 消息窗口限制 50 条（`MemoryConfig` 配置）
- 记忆存储：优先 Redis，无 Redis 时回退到内存

### 2.3 工具调用 Agent 模块 (ToolAgent)

**涉及文件**：
- `agent/ToolAgent.java`
- `controller/AgentController.java`
- `tool/CalculatorTool.java`
- `tool/DateTimeTool.java`
- `tool/TranslateTool.java`
- `tool/DocParseTool.java`
- `tool/DbQueryTool.java`

**功能说明**：
- 工具调用对话无记忆模式（`POST /api/v1/agent/chat`）：AI 自主决定是否调用工具
- 工具调用对话带记忆模式（`POST /api/v1/agent/chat/memory`）：多轮上下文 + 工具调用
- 指定工具对话（`POST /api/v1/agent/chat/specific`）：只启用用户指定的工具集

**5 个工具详解**：

| 工具 | 能力 | 实现方式 |
|------|------|---------|
| CalculatorTool | 数学表达式计算、百分比、单位换算 | JS ScriptEngine |
| DateTimeTool | 当前时间、星期几、日期差、日期推算 | java.time API |
| TranslateTool | 多语言翻译、语言检测 | 内部调用大模型 |
| DocParseTool | 读取本地 txt/md 文件、列出文件 | NIO Files API，含路径穿越防护 |
| DbQueryTool | 员工查询、部门查询、统计 | 内存模拟数据 |

**执行逻辑**：
```
用户消息 → ChatClient.prompt()
  → .tools(dateTimeTool, calculatorTool, translateTool, docParseTool, dbQueryTool)
  → AI 分析是否需要调用工具
    → 需要：执行工具方法 → 结果回传给 AI → AI 整合生成最终回复
    → 不需要：AI 直接回答
  → 返回 AgentResponse（含 conversationId、reply、traceId、model、agentType）
```

**安全设计**：
- DocParseTool：防路径穿越（禁止 `..` `/` `\`），文件大小限制 10KB
- CalculatorTool：正则校验只允许 `[0-9+\\-*/().%\\s]`
- 所有工具方法都有 try-catch 错误处理

### 2.4 RAG 知识库模块 (RagAgent + DocumentService + RetrievalService)

**涉及文件**：
- `agent/RagAgent.java`
- `controller/KnowledgeController.java`
- `rag/DocumentService.java`
- `rag/RetrievalService.java`
- `config/EmbeddingConfig.java`
- `config/QdrantConfig.java`

**功能说明**：

**文档管理**：
- 上传文档（`POST /api/v1/knowledge/upload`）：支持 .txt / .md / .pdf / .docx
- 批量加载目录（`POST /api/v1/knowledge/load-directory`）
- 文档列表（`GET /api/v1/knowledge/documents`）
- 删除文档（`DELETE /api/v1/knowledge/documents/{fileName}`）

**RAG 问答**：
- 自动 RAG 问答（`POST /api/v1/knowledge/ask`）：QuestionAnswerAdvisor 自动检索+回答
- 手动 RAG 问答（`POST /api/v1/knowledge/ask/manual`）：手动控制检索和拼接
- 流式 RAG 问答（`GET /api/v1/knowledge/stream`）：SSE 流式推送
- 纯检索（`GET /api/v1/knowledge/search`）：调试用，只返回检索片段
- 知识库状态（`GET /api/v1/knowledge/status`）

**文档处理流水线**：
```
上传文件 → 校验文件类型(.txt/.md/.pdf/.docx) 
→ 读取内容(TextReader/PagePdfDocumentReader/POI docx) 
→ TokenTextSplitter 切片(chunkSize=800, overlap=100) 
→ EmbeddingModel 向量化(DashScope text-embedding-v4) 
→ Qdrant 向量存储 
→ SimpleVectorStore 持久化到本地 JSON
```

**RAG 问答执行逻辑**：
```
用户问题 
→ RetrievalService.retrieve()
  → EmbeddingModel 将问题转为向量
  → Qdrant 向量库相似度检索(topK=5, threshold=0.5)
  → 返回相关文档片段
→ 构建 Prompt：「参考资料 + 用户问题」
→ ChatClient 生成回答
→ 返回 KnowledgeResponse（含 answer、sources、retrievedChunks）
```

**向量检索过滤**：
- 全局知识库检索：`userId == 'system' && enabled == true`
- 用户个人检索：`userId == '<username>' && enabled == true`

### 2.5 任务规划 Agent 模块 (PlanningAgent)

**涉及文件**：
- `agent/PlanningAgent.java`
- `controller/PlanningController.java`

**功能说明**：
- 任务规划并执行（`POST /api/v1/planning/execute`）：AI 判断是否需要规划，复杂任务拆解多步执行
- 仅规划不执行（`POST /api/v1/planning/plan-only`）：只返回步骤列表
- 流式规划（`GET /api/v1/planning/stream`）：SSE 流式展示规划过程
- 全能 Agent 入口（`POST /api/v1/planning/chat`）：整合记忆+工具的对话

**规划执行流程**：
```
用户任务 → AI 生成规划 JSON（含 needPlanning、steps、taskSummary）
  → needPlanning == false：直接返回 directAnswer
  → needPlanning == true：
    → autoExecute == false：只返回步骤列表
    → autoExecute == true：逐步骤执行
      → 每个步骤检查是否需要工具（toolNeeded）
        → 需要工具：executeWithTools(stepPrompt, toolName)
        → 不需要：baseChatClient 直接执行
      → 上下文累积：前面步骤的结果注入后续步骤
      → generateFinalAnswer()：汇总所有步骤结果生成最终回答
  → 返回 PlanningResponse（含 taskSummary、steps、finalAnswer、totalTimeMs）
```

**规划 JSON 格式示例**：
```json
{
  "needPlanning": true,
  "taskSummary": "读取文件并总结然后翻译",
  "steps": [
    { "stepNumber": 1, "description": "读取sample.md文件", "toolNeeded": "doc" },
    { "stepNumber": 2, "description": "总结核心内容", "toolNeeded": "无" },
    { "stepNumber": 3, "description": "翻译成英文", "toolNeeded": "translate" }
  ]
}
```

**兜底机制**：JSON 解析失败时，回退到 FullAgent 直接执行

### 2.15 LearnAgent 个人学习系统

这是项目最重要的特色模块，位于 `learn/` 包下。

#### 2.15.1 ReAct 推理引擎 (ReActEngine)

**涉及文件**：
- `learn/service/ReActEngine.java`
- `learn/controller/LearnChatController.java`

**ReAct 推理循环**（Thought → Action → Observation）：
```
用户学习问题
  → Thought: "我会先检索个人知识库和导入资料，再基于来源生成回答"
  → Action: search_knowledge (检索用户笔记 + 全局知识库)
  → Observation: "检索到 N 条相关来源" / "未检索到直接相关来源"
  → Final Answer: AI 基于来源生成学习回答
  → suggestedActions: 建议生成测验 / 闪卡 / 保存笔记
```

**核心 API**：
| 接口 | 功能 |
|------|------|
| `POST /api/v1/learn/chat` | 同步 ReAct 对话 |
| `GET /api/v1/learn/chat/stream` | SSE 流式 ReAct 对话 |
| `GET /api/v1/learn/chat/traces/{traceId}` | 获取推理链路详情 |
| `GET /api/v1/learn/chat/traces` | 按 sessionId 列出推理链路 |
| `POST /api/v1/learn/chat/interventions` | 用户干预：在推理中插入补充信息 |
| `POST /api/v1/learn/chat/stop` | 停止正在运行的推理 |

**知识检索策略**（ReActEngine.retrieveContext）：
```
1. 优先搜索用户个人笔记（noteService.search）
2. 如果笔记搜索无结果 → 搜索全局向量库（retrievalService.retrieveForUser）
3. 结果合并为来源引用列表（ReActSourceRefDocument）
```

**兜底模型**：主模型失败 → 备用模型 → 本地规则回答（localAnswer）

**可配置参数**（application.yml）：
- `max-iterations`: 6（最大推理迭代）
- `step-timeout-seconds`: 15（步骤超时）
- `stream-heartbeat-seconds`: 20（流式心跳）
- `default-strategy`: auto（默认策略）

#### 2.15.4 闪卡复习模块 (FlashcardService)

**涉及文件**：
- `learn/service/FlashcardService.java`
- `learn/controller/FlashcardController.java`

**功能说明**：
- 基于笔记生成闪卡（正面=问题，背面=答案）
- SM-2 间隔重复算法调度复习
- 根据评分质量调整间隔天数和难度因子

**SM-2 算法实现**：
```
quality >= 3（记住了）：
  reviewCount == 1 → nextInterval = 1天
  reviewCount == 2 → nextInterval = 6天
  else → nextInterval = round(previousInterval * easeFactor)
  easeFactor = max(1.3, EF + 0.1 - (5-q) * (0.08 + (5-q) * 0.02))

quality < 3（忘了）：
  reviewCount = 0, lapseCount++
  nextInterval = 1天（重新开始）
  easeFactor = max(1.3, EF - 0.2)
```

**评分影响掌握度变化**：
| 评分(quality) | masteryDelta | 含义 |
|---------------|-------------|------|
| 5 | +5 | 完美回忆 |
| 4 | +3 | 犹豫后正确 |
| 3 | +1 | 困难但正确 |
| 2 | -2 | 错误但答案熟悉 |
| 1 | -4 | 完全忘记 |
| 0 | -6 | 完全没印象 |

#### 2.15.5 知识图谱模块 (GraphService)

**涉及文件**：
- `learn/service/GraphService.java`
- `learn/controller/GraphController.java`

**功能说明**：
- 全局图谱（所有笔记按分类/标签关联）
- 围绕笔记的子图（相关笔记 + 共享标签笔记）

**图谱结构**：
```
nodes（节点）:
  - note 节点（笔记）：含 masteryLevel、category、标签组
  - tag 节点（标签）
  - category 节点（分类）

edges（边）:
  - note → tag（标签关联，weight=0.6）
  - note → category（分类关联，weight=0.3）
  - note → note（参考关联，weight=0.8）
```

**节点颜色分组**：
| 状态 | masteryLevel | 颜色组 |
|------|-------------|--------|
| 未学习 | 0 | "new" |
| 薄弱 | < 40 | "weak" |
| 学习中 | 40-79 | "learning" |
| 已掌握 | >= 80 | "mastered" |

#### 2.15.6 学习统计模块 (StudyService)

**涉及文件**：
- `learn/service/StudyService.java`

**功能说明**：

- 总体统计：笔记总数、导入文档数、测验数、闪卡数、已掌握/薄弱笔记数
- 最近主题列表
- 薄弱点分析（masteryLevel < 40 的笔记）
- 学习活动记录（StudyRecordEntity - MySQL）

**薄弱点分析逻辑**：
```
查询所有未归档笔记 → 过滤 masteryLevel < 40 → 按掌握度升序 
→ 严重度判断（<20: high, >=20: medium）

```

**掌握度变化应用**：
```
应用 masteryDelta → 限制在 [0, 100] → 保存笔记 → 记录 mastery_update 活动
```

## 四、基础设施与横切关注点

### 4.1 全链路追踪 (TraceContext)

**涉及文件**：
- `common/context/TraceContext.java` — ThreadLocal 存储 traceId
- `common/context/TraceContextFilter.java` — 从请求头 `X-Trace-Id` 获取或生成
- `common/context/TraceContextMdcFilter.java` — 注入 SLF4J MDC

**实现**：每个请求生成唯一 traceId，贯穿整个调用链，Agent 返回时携带 traceId，便于日志关联和问题排查。

### 4.2 全局异常处理 (GlobalExceptionHandler)

```
BizException        → 400 业务异常
NonTransientAiException → 500 AI 模型调用异常
MethodArgumentNotValidException → 400 参数校验失败
ConstraintViolationException → 400 约束校验失败
AccessDeniedException → 403 权限不足
Exception（兜底）        → 500（dev环境返回详情，生产返回通用消息）
```

### 4.3 统一响应格式 (R<T>)

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... },
  "timestamp": 1700000000000,
  "traceId": "xxx-xxx-xxx"
}
```

### 4.4 数据库设计

| 数据库 | 用途 | 主要表/集合 |
|--------|------|-----------|
| MySQL (JPA) | 用户、聊天历史、会话、Agent 调用记录、学习记录 | users, chat_history, chat_session, agent_invocation, study_records |
| MongoDB | 学习模块文档存储 | knowledge_notes, flashcards, quizzes, react_traces |
| Redis | 会话记忆持久化 | chat memory (key-value) |
| Qdrant / SimpleVectorStore | 向量存储 | 文档切片向量 |

### 4.5 AI 模型配置

**主模型**：DeepSeek（通过 OpenAI 兼容协议接入）
- 配置项：`spring.ai.openai.base-url`、`DEEPSEEK_API_KEY`
- 默认模型：`deepseek-v4-flash`

**兜底模型**：Qwen（通过阿里云 DashScope 兼容接入）
- 配置项：`qwen.api-base-url`、`QWEN_API_KEY`
- 默认模型：`qwen3.6-plus`

**Embedding 模型**：DashScope text-embedding-v4
- 用于文本向量化，支持 2048 维向量

**切换逻辑**：主模型重试 1 次失败 → 备用模型重试 1 次失败 → 抛出异常

### 4.6 网络配置

- 连接超时：8秒
- 读取超时：25秒
- 重试间隔：200ms

## 六、关键设计模式与亮点

1. **模型兜底切换**：主模型 + 备用模型双重保障，ChatClientConfig 通过独立 Bean 隔离
2. **多 Agent 协作**：ChatAgent（对话）、ToolAgent（工具）、RagAgent（检索）、PlanningAgent（规划）、FullAgent（全能）各司其职
3. **ReAct 推理模式**：Thought → Action → Observation 循环，可追踪、可干预、可停止
4. **SM-2 间隔重复**：标准记忆算法实现，自动调整复习间隔和难度因子
5. **向量双检索策略**：优先笔记搜索（关键词），无结果再向量检索（语义相似度）
6. **全链路 TraceId**：ThreadLocal + Filter + MDC，从请求到响应全程可追踪
7. **审计全覆盖**：每次 Agent 调用自动记录输入/输出/模型/耗时/状态
8. **前端 SSE 流式消费**：fetch ReadableStream 解析 JSON 事件流，实时更新 UI 状态
9. **文档多格式支持**：txt/md/pdf/docx 统一通过不同 Reader 解析
10. **配置外部化**：模型参数、超时、重试、切片参数全部可配置
