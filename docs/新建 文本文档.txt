一、任务理解
目标：为 LearnAgent 项目制定学习优化方案，包括现有功能改进和新增功能规划

约束：只生成规划文档，不编写代码

输出格式：结构化的优化方案文档，包含改进点、新增功能、实施优先级、技术方案

二、现有问题诊断
2.1 学习体验层面的问题
问题	现状	影响
推理策略选择不够智能	用户手动选择 auto/retrieve_first/explain_first/quiz_first	用户负担重，选择不当影响学习效果
学习路径缺失	只有单次对话，没有系统化的学习路径规划	学习碎片化，缺乏系统性

前端交互不够沉浸	ReAct 面板和对话区分离	学习注意力分散

四、详细优化方案
O1：智能推理策略优化
问题描述
当前用户需要手动选择推理策略（auto/retrieve_first/explain_first/quiz_first），这增加了用户负担，且选择不当会影响学习效果。

优化目标
实现策略自动选择，系统根据问题内容自动判断最优推理路径。

优化方案
方案一：关键词规则匹配（轻量级）

问题特征	自动策略	示例
包含"是什么"、"定义"、"概念"	explain_first	"梯度下降是什么？"
包含"区别"、"对比"、"比较"	retrieve_first + 对比模板	"SGD 和 Adam 有什么区别？"
包含"测验"、"自测"、"检验"	quiz_first	"帮我测验一下优化器知识"
包含"复习"、"回顾"、"记得"	retrieve_first + 复习模板	"复习一下反向传播"
包含"薄弱"、"不懂"、"困惑"	explain_first + 深度解释	"我对反向传播很困惑"
包含具体知识点名称	retrieve_first	"讲一下 SGD 的原理"
其他	auto	默认策略
方案二：LLM 策略判断（智能级）

在 ReAct 启动前，先用轻量模型（或快速调用）判断问题类型：

F2：复习优先级智能调度

当前 SM-2 算法只基于复习历史，不考虑掌握度。优化为：


Plain Text

复习优先级 = 基础优先级 × 掌握度权重 × 超期权重

基础优先级：SM-2 算法计算
掌握度权重：掌握度 < 40% → 权重 2.0；40-60% → 1.5；> 60% → 1.0
超期权重：超期天数 × 0.1 + 1.0

示例：
- 反向传播：基础优先级 5，掌握度 35%，超期 2 天
  → 最终优先级 = 5 × 2.0 × 1.2 = 12（最高优先）

F1：专业化 Agent 定义

Agent	专注场景	核心能力	工具集
ExplainAgent	概念解释	深度解释、类比说明、图解辅助	KnowledgeTool, WebSearchTool
CompareAgent	知识对比	对比分析、差异提炼、选择建议	KnowledgeTool, find_related
QuizAgent	测验检验	测验生成、答案评估、错题分析	QuizTool, LearningTool
ReviewAgent	复习巩固	间隔复习、闪卡翻转、费曼检验	QuizTool, FlashcardTool
ExploreAgent	知识探索	Web 搜索、新知识沉淀、关联发现	WebSearchTool, KnowledgeTool
PlanAgent	学习规划	路径生成、进度追踪、调整建议	LearningTool, GraphService
F2：Agent 协同流程


Plain Text

用户问题："帮我系统学习深度学习优化器"

PlanAgent 接收：
1. 分析用户目标（系统学习）
2. 检索知识库，发现已有笔记和缺口
3. 生成学习路径（4 个阶段）
4. 返回路径 + 每阶段推荐 Agent

阶段1（基础概念）→ ExplainAgent 执行
阶段2（进阶对比）→ CompareAgent 执行
阶段3（测验检验）→ QuizAgent 执行
阶段4（复习巩固）→ ReviewAgent 执行

协同数据传递：
- 每阶段完成后，结果传递给下一阶段
- PlanAgent 监控整体进度
- 异常时（测验得分低），PlanAgent 调整路径

Plain Text

用户问题 → AgentRouter → 选择最优 Agent

路由规则：
- 包含"解释"、"是什么" → ExplainAgent

- 包含"测验"、"自测" → QuizAgent
- 包含"复习"、"回顾" → ReviewAgent
- 包含"搜索"、"查找资料" → ExploreAgent
- 包含"学习计划"、"系统学习" → PlanAgent
- 复杂问题 → PlanAgent 协调多 Agent
